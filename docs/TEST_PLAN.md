# Daisy 测试计划

## 自动化门禁

每次提交必须通过：

1. Windows 仓库结构、资源 JSON、Info.plist、命名和秘密扫描。
2. macOS XcodeGen 工程生成。
3. iOS Simulator Debug 编译。
4. XCTest 单元测试。
5. XCUITest 启动、手动记账，以及 AI URL/模型保存、返回和重开状态流程。
6. iphoneos Release 无签名构建和 IPA 打包。

## 单元测试覆盖

- 金额字符串、最小货币单位和四舍五入。
- Base URL 拼接、HTTPS 限制、局域网与个人组网地址。
- Markdown 包裹 JSON 的提取和非法模型输出拒绝。
- 金额、币种、分类、置信度边界、大额和 OCR 冲突验证。
- 普通账单置信度恰好达到阈值时自动入账；模型普通 warning 和视觉测试状态不额外阻断。
- 商户缺失、分类纠正、异常时间、OCR 金额冲突、大额与转账必须进入确认。
- 商户标准化、OCR 金额候选。
- CSV 转义、JSON 备份恢复。
- JSON 备份的周期提醒、时间元数据、旧版 schema 兼容和逻辑重复预算校验。
- 账户余额中的收入、支出、退款、转入和转出计算。
- JSON 备份的账户引用、重复 ID、非法金额和未来 schema 拒绝。
- 资产/负债汇总口径、外币排除、转账不改变总财富、估值和余额校准不改变月度收支。
- JSON 资产、估值历史、余额校准的完整往返；旧 schema 缺少新增字段仍可读取；关闭“计入净资产”仍是合法数据；孤立历史记录拒绝恢复。
- AI 自动兼容降级、5xx 单次重试和多种消息内容结构。
- AI 配置的未配置/已配置/已验证状态、字段规范化和 UserDefaults 往返。
- 同一截图失败后重试可更新待确认结果，成功后清理旧失败记录。
- App Intent 元数据必须限定 `public.image` 并自动连接上一步快捷指令输出。

## 必做实机矩阵

在目标 iPhone 上逐项记录结果：

| 场景 | 预期 |
|---|---|
| App 未启动时运行快捷指令 | Intent 收图并给出结果横幅 |
| 签名实机保存并重开 AI 配置 | URL、模型和 API Key 均保持；Key 不显示在设置摘要 |
| 快捷指令仅含 Daisy 动作 | 明确提示未收到截屏，不产生账单 |
| “截屏 → Daisy”参数连接 | Daisy 的“付款截图”显示为上一步“截屏”变量 |
| App 前台/后台运行 | 不重复创建账单 |
| 微信/支付宝付款成功页 | 金额、商户、方向正确 |
| 多金额/优惠页面 | 不确定时进入确认，不自动误记 |
| 退款、转账、信用卡还款 | 不作为普通消费自动入账 |
| 总览预算下钻 | 返回总览而不是设置 Tab |
| 设置预算入口 | 返回设置，并默认当前月份 |
| 分析分类下钻 | 返回分析，账单根 Tab 不残留隐式筛选 |
| 相同截图连续运行 | 只保留一条正式账单 |
| 断网、超时、401、404、429、5xx | 错误可理解，不产生半条账单 |
| 本地 Ollama Windows 休眠 | 快速失败并保留手动入口 |
| 深色模式、最大动态字体、VoiceOver | 核心信息和按钮可用 |
| AltStore 两次 7 天刷新 | App、Keychain、账本、快捷指令仍有效 |
| JSON 导出、清空、恢复 | 金额、时间、来源和分类一致 |
| 资产导出、清空、恢复 | 账户分组、净资产开关、估值历史和校准记录一致 |
| 资产估值更新 | 净资产变化；月度收入、支出和预算不变 |
| 账户余额校准 | 账户余额变化；不新增收入或支出账单 |
| 外币账户/资产 | 明细保留；人民币汇总排除并显示提示 |

## 交付门槛

- macOS CI 全绿。
- 无签名 IPA 工作流成功。
- 目标 iPhone 完成背面轻点端到端测试。
- 至少 200 张脱敏付款截图回归集中，金额完全匹配率达到 99%。
- 没有 P0/P1 崩溃、数据损坏、重复入账或 API Key 泄露问题。
