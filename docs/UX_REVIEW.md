# Daisy 用户体验审查与演进路线

审查日期：2026-07-19

## 产品画像

Daisy 面向希望轻量、私密、快速掌握个人现金流的 iPhone 用户。它不追求堆叠复杂财务功能，而应稳定完成三个核心任务：

1. 用尽可能少的操作记下一笔账。
2. 随时确认“钱花到哪里、这个月还剩多少”。
3. 发现错误后能快速查看、修改或删除，不让数据变成孤岛。

建议长期坚持的体验承诺：**3 秒开始记账，10 秒完成核对，任何账单一触即达。**

## 竞争产品启示

本次调研只采用产品官方页面和帮助中心作为功能依据：

- Monarch 将交易复核放在 Dashboard 和 Transactions 两条路径中，并允许进入交易后继续编辑或标记状态。
- Copilot 把最近交易、待复核交易、筛选和交易详情设计为连续流程，减少用户在不同页面间寻找入口的成本。
- YNAB 的报告强调从汇总继续下钻到分类和明细，让图表承担解释与行动入口，而不只是展示。

这些产品共同证明：用户喜欢的不是“功能数量”，而是信息之间有连续关系——汇总可以下钻，列表可以查看，详情可以修正，修正后立即反馈。

## 本轮已完成

本轮接管审查发现，上一轮虽然增加了大量入口，但总览下钻通过切换 Tab 实现，造成返回层级丢失；同时账单筛选会跨会话残留。已按“主 Tab 是工作域、NavigationStack 是任务流”的原则修正：

- 总览进入预算、待确认、日期账单后，返回仍回到总览。
- 分析进入分类账单后，返回仍回到分析，账单页不再被写入隐式日期/分类筛选。
- 设置进入预算始终从当前月份开始，与总览查看历史月份的入口语义分离。
- 快速记账优先复用同商户历史分类与账户，无法判断时使用“其他支出”。
- 周期提醒随 JSON 备份恢复，旧版备份仍可读取；保存失败会回滚模型和通知状态。
- 退款会抵扣总预算已用金额，分类预算暂按原始分类支出统计并在界面明示限制。

- 仪表盘“最近账单”中的每一行可直接进入账单详情，同时保留“查看全部”。
- 统一金额组件，将正负号与货币金额按基线和固定符号宽度排版，修复金额变长后的视觉错位。
- 账单详情新增编辑入口，复用记账表单并提供保存反馈。
- 删除详情账单前增加确认，账单列表关闭全滑直接删除，降低误操作风险。
- 删除后提供 5 秒撤销入口，并用完整账单快照恢复原始字段。
- 区分“没有任何账单”和“筛选后没有结果”，并提供明确行动按钮。
- 账单页保留类型筛选；日期和分类筛选由总览/分析下钻上下文承载，返回后自动结束。
- 月度预算卡片整体可点击，并直接打开当前月份的预算设置。
- 支出趋势图支持选择日期、查看当天金额并进入当天账单。
- 隐藏金额时同步保护 VoiceOver 文案，避免视觉隐藏但辅助功能仍泄露金额。
- 新增 UI 测试，覆盖“最近账单 → 详情 → 编辑”核心链路。

## 体验原则

### 连续而非孤立

所有摘要、图表和列表都应回答“接下来用户能做什么”。可展示的数据原则上应支持下钻；可发现的问题应就地提供修正入口。

### 快速但可撤回

高频操作应少步骤，破坏性操作应防误触。后续应优先为删除增加撤销能力，而不是增加更多确认弹窗。

### 信息层级稳定

金额始终右对齐并使用等宽数字；商户是第一信息，分类与时间是第二信息；颜色只辅助区分收入、支出和风险，不作为唯一提示。

### 隐私覆盖完整

隐藏金额需要覆盖仪表盘、列表、详情、图表标签、VoiceOver、应用切换器快照和未来的小组件。

### 原生与克制

优先使用 NavigationStack、List、Form、Sheet、Menu、系统材质和 SF Symbols。动效用于解释状态变化，不用于制造额外视觉负担，并尊重 Reduce Motion。

## 后续优先级

### P0：形成闭环（已完成）

- [x] 删除后提供短时撤销。
- [x] 图表点选后展示日期、金额，并可进入对应账单列表。
- [x] 预算卡片直接进入当前月份的预算设置。
- [x] 在账单页提供类型筛选的可见状态；日期/分类下钻保持来源页面上下文。

### P1：降低记账成本（已完成）

- [x] 记住用户上一次使用的账户和各类型常用分类，但不覆盖明确选择。
- [x] 为商户提供本地历史建议，并根据历史记录带出分类和账户。
- [x] 支持复制一笔旧账生成新账，复制时默认使用当前时间。
- [x] 在仪表盘提供条件式待确认入口，并支持逐项核对识别结果。

### P2：提升理解能力（已完成）

- [x] 增加近 7 天/月度支出对比和较高支出提示，避免复杂财务术语。
- [x] 支持本地周期账单提醒，通知不显示商户或金额，点击后仍需手动确认入账。
- [x] 在主应用目标内提供“快速记一笔”App Intent，不新增后端或额外 App ID。
- [x] 分类排行可下钻至对应月份与分类的账单列表。
- [x] 下钻返回保留来源 Tab，不把任务流伪装成 Tab 切换。
- [x] JSON 备份包含周期提醒和账单/预算时间元数据。
- [x] 新增“资产”工作区：总览展示净资产、总存款、总资产和负债，资产页面按金融账户、其他资产、其他负债和归档项目分组。
- [x] 资产入口保持来源上下文：总览进入资产后返回总览，设置进入资产后返回设置；不通过切换 Tab 制造错误的返回路径。
- [x] 余额校准和估值更新明确提示不会生成账单，估值历史可在详情页查看；关闭“计入净资产”保留数据但从汇总排除。
- [x] 外币项目不强行换算，使用清晰提示解释人民币汇总边界。

系统小组件需要额外扩展目标和 App ID，与当前免费签名尽量减少 App ID 的约束冲突，因此本阶段不引入 Widget Extension；快捷入口由主目标 App Intent 提供。

### 当前不虚假承诺的边界

- 分类预算还没有“退款关联原账单”的数据模型，因此退款只抵扣总预算，不冒充已完成分类回冲。
- JSON 备份当前是明文文件，不是加密备份；界面会明确提示用户自行保护导出文件。
- 实机通知、Face ID、快捷指令和免费签名续签仍需在目标 iPhone 上验收，CI 不能替代这些外部系统测试。
- 当前资产页面只做手动估值，不连接行情、汇率或第三方资产账户；因此不会把波动伪装成收入，也不承诺实时投资收益。
