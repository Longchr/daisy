# Daisy 用户体验审查与演进路线

审查日期：2026-07-18

## 产品画像

Daisy 面向希望轻量、私密、快速掌握个人现金流的 iPhone 用户。它不追求堆叠复杂财务功能，而应稳定完成三个核心任务：

1. 用尽可能少的操作记下一笔账。
2. 随时确认“钱花到哪里、这个月还剩多少”。
3. 发现错误后能快速查看、修改或删除，不让数据变成孤岛。

建议长期坚持的体验承诺：**3 秒开始记账，10 秒完成核对，任何账单一触即达。**

## 竞争产品启示

本次调研只采用产品官方页面和帮助中心作为功能依据：

- Monarch 将交易复核放在 Dashboard 和 Transactions 两条路径中，并允许进入交易后继续编辑或标记状态。
- Copilot 把最近交易、待复核交易、筛选和交易详情设计为连续流程，减少用户在不同页面间寻找入口的成本。
- YNAB 的报告强调从汇总继续下钻到分类和明细，让图表承担解释与行动入口，而不只是展示。

这些产品共同证明：用户喜欢的不是“功能数量”，而是信息之间有连续关系——汇总可以下钻，列表可以查看，详情可以修正，修正后立即反馈。

## 本轮已完成

- 仪表盘“最近账单”中的每一行可直接进入账单详情，同时保留“查看全部”。
- 统一金额组件，将正负号与货币金额按基线和固定符号宽度排版，修复金额变长后的视觉错位。
- 账单详情新增编辑入口，复用记账表单并提供保存反馈。
- 删除详情账单前增加确认，账单列表关闭全滑直接删除，降低误操作风险。
- 删除后提供 5 秒撤销入口，并用完整账单快照恢复原始字段。
- 区分“没有任何账单”和“筛选后没有结果”，并提供明确行动按钮。
- 账单页持续显示日期和类型筛选状态，并可分别一键清除。
- 月度预算卡片整体可点击，并直接打开当前月份的预算设置。
- 支出趋势图支持选择日期、查看当天金额并进入当天账单。
- 隐藏金额时同步保护 VoiceOver 文案，避免视觉隐藏但辅助功能仍泄露金额。
- 新增 UI 测试，覆盖“最近账单 → 详情 → 编辑”核心链路。

## 体验原则

### 连续而非孤立

所有摘要、图表和列表都应回答“接下来用户能做什么”。可展示的数据原则上应支持下钻；可发现的问题应就地提供修正入口。

### 快速但可撤回

高频操作应少步骤，破坏性操作应防误触。后续应优先为删除增加撤销能力，而不是增加更多确认弹窗。

### 信息层级稳定

金额始终右对齐并使用等宽数字；商户是第一信息，分类与时间是第二信息；颜色只辅助区分收入、支出和风险，不作为唯一提示。

### 隐私覆盖完整

隐藏金额需要覆盖仪表盘、列表、详情、图表标签、VoiceOver、应用切换器快照和未来的小组件。

### 原生与克制

优先使用 NavigationStack、List、Form、Sheet、Menu、系统材质和 SF Symbols。动效用于解释状态变化，不用于制造额外视觉负担，并尊重 Reduce Motion。

## 后续优先级

### P0：形成闭环（已完成）

- [x] 删除后提供短时撤销。
- [x] 图表点选后展示日期、金额，并可进入对应账单列表。
- [x] 预算卡片直接进入当前月份的预算设置。
- [x] 在账单页提供日期与类型的可见筛选状态。

### P1：降低记账成本

- 记住用户上一次使用的账户和常用分类，但不覆盖明确选择。
- 为商户提供本地历史建议，并根据历史记录建议分类。
- 支持复制一笔旧账生成新账，适合重复但非固定周期的消费。
- 对 AI 识别结果提供清晰的“待确认”队列和逐项修正体验。

### P2：提升理解能力

- 增加周/月同比和异常支出提示，避免引入复杂财务术语。
- 支持轻量周期账单提醒，但保持手动确认入账。
- 在不泄露隐私的前提下提供系统小组件和 App Intent 快速入口。
